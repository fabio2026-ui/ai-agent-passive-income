# 🔧 彻底修复端口冲突

========================================
❌ 问题：端口仍被占用
========================================

你执行了：
- kill -9 62863 → 成功（进程不存在了）
- 但重启仍然失败 → 端口还被占用

**原因：** 可能有其他进程占用了9999端口

========================================
🔧 解决方案
========================================

## 步骤1: 查找占用端口的进程

```bash
lsof -i :9999
```

会显示类似：
```
COMMAND   PID    USER   FD   TYPE
Python   xxxxx  fabiofu  ...
```

## 步骤2: 杀死所有占用进程

```bash
kill -9 $(lsof -t -i :9999)
```

或者先查看再杀死：
```bash
# 查看PID
lsof -i :9999 | grep LISTEN | awk '{print $2}'

# 杀死PID（把xxxx换成实际的数字）
kill -9 xxxx
```

## 步骤3: 验证端口已释放

```bash
lsof -i :9999
```

应该**没有任何输出**

## 步骤4: 重新启动服务

```bash
python3 ~/ai_secure_v2.py &
```

========================================
✅ 一键修复命令
========================================

复制粘贴执行：

```bash
# 杀死所有占用9999端口的进程
kill -9 $(lsof -t -i :9999) 2>/dev/null; sleep 2; python3 ~/ai_secure_v2.py &
```

然后检查：
```bash
lsof -i :9999
```
