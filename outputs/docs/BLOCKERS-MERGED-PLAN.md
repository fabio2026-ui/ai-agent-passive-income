## 二、账号合并方案（减少重复注册）

### 方案A：统一支付收款账号

**现状问题**:
- EU API → 需要Stripe
- Notion模板 → 需要Gumroad（或Stripe）
- React Apps → 应用商店收款

**合并策略**: **只用Stripe一个收款渠道**

| 项目 | 原方案 | 合并后方案 | 优势 |
|------|--------|------------|------|
| EU API | Stripe订阅 | Stripe订阅 | ✅ 不变 |
| Notion模板 | Gumroad | Stripe Payment Links | ✅ 免Gumroad注册 |
| 数字产品 | 多平台 | Stripe Checkout | ✅ 统一后台 |

**操作方式**:
- Stripe支持一次性付款（Payment Links）
- 可以卖模板、卖服务、收订阅
- 一个Dashboard管理所有收入

**节省**: 少注册1个Gumroad账号
