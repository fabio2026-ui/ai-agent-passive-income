#### 测试方法

**方法1: 假门测试 (Fake Door)**
```
步骤:
1. 创建产品 landing page
2. 投放小额广告 ($100-300)
3. 看点击率/预约率
4. >5%转化 = 需求验证成功

成本: $200-500/次
时间: 1-2周
```

**方法2: 预售测试**
```
步骤:
1. 产品概念 + 预售价格
2. "即将推出，预售价$X"
3. 看是否有人付钱预订
4. 有人付钱 = 需求真实

成本: $100-300 (页面+广告)
时间: 2-3周
```

**方法3: MVP快速测试**
```
步骤:
1. 开发最简单可用版本
2. 找5-10个用户测试
3. 收集反馈
4. NPS>50 = 产品有价值

成本: $500-2000
时间: 3-4周
```

📊 风险管理
========================================

## 风险控制原则
1. ** never投入全部资金** - 保留40%应急
2. **先验证再投入** - 小成本测试
3. **分散验证** - 同时测试2-3个，不押注1个
4. **及时止损** - 验证失败立即停，不追加投入
5. **保持Bootstrap现金流** - 即使大项目失败，还有收入

## 应急预案

### 场景A: Bootstrap收入不达标
**应对**:
- 延长Phase 1，继续优化
- 尝试更多平台/服务类型
- 降低生活成本，延长跑道

### 场景B: 验证全部失败
**应对**:
- 回到Phase 1继续Bootstrap
- 重新研究趋势
- 保留资金，等待更好机会

### 场景C: 大项目投入后失败
**应对**:
- 立即停止，不追加投入
- 保留剩余资金
- 回到Bootstrap重建现金流
