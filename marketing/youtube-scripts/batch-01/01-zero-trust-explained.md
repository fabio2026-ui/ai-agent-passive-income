## 📋 CONTENT STRUCTURE

### Section 1: The Problem (0:45 - 2:00)

**What to cover:**
- Traditional perimeter-based security ("trust but verify")
- Why it failed: Cloud migration, remote work, BYOD policies
- Real breach examples where internal threats were the vector
- Statistics: Average time to detect a breach (287 days)

**Key Points:**
- The old model assumes everything inside the network is safe
- 80% of breaches involve compromised credentials
- Once inside, attackers have free reign

### Section 2: What is Zero Trust? (2:00 - 4:00)

**Definition:**
"Never trust, always verify"

**Core Principles:**
1. **Verify Explicitly** - Authenticate and authorize based on all available data points
2. **Use Least Privilege Access** - Just-in-time and just-enough-access (JIT/JEA)
3. **Assume Breach** - Minimize blast radius, segment access, verify end-to-end encryption

**Visual:**
- Show architecture diagram: Identity → Device → Application → Data
- Each layer requires verification

### Section 3: Zero Trust Architecture Components (4:00 - 6:30)

**Identity:**
- Multi-factor authentication (MFA)
- Conditional access policies
- Risk-based authentication

**Devices:**
- Device health verification
- Endpoint detection and response (EDR)
- Mobile device management (MDM)

**Applications:**
- Application-level access controls
- API security gateways
- Cloud access security broker (CASB)

**Data:**
- Data classification and labeling
- Encryption at rest and in transit
- Data loss prevention (DLP)

**Network:**
- Micro-segmentation
- Software-defined perimeter (SDP)
- Encrypted connections everywhere

### Section 4: Real-World Implementation (6:30 - 8:00)

**Step-by-Step Guide:**
1. **Start with identity** - Enable MFA everywhere (free with Google/Microsoft)
2. **Map your data** - Know what you have and where it is
3. **Segment your network** - VLANs, private subnets, VPN alternatives
4. **Monitor everything** - SIEM, logging, behavioral analytics
5. **Automate responses** - Conditional access, automated remediation

**Tools mentioned:**
- Microsoft Azure AD / Entra ID
- Google BeyondCorp
- Okta
- Cloudflare Zero Trust
- Zscaler

### Section 5: Common Mistakes (8:00 - 9:00)

**Pitfalls to avoid:**
- Trying to do everything at once
- Forgetting about legacy systems
- Ignoring user experience (friction = shadow IT)
- Lack of executive buy-in
- Treating Zero Trust as a product, not a strategy
