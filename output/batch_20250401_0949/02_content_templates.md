### 使用说明

1. **填空指南**：
   - `[公司名]`：使用正式公司名
   - `[创始人引言]`：保持真实，不要过度包装
   - `[年份]`：用真实时间节点
   - `[用户数量]`：用最新数据，定期更新

2. **调整建议**：
   - B2B公司：强调专业背景和客户案例
   - B2C公司：强调情感连接和用户体验
   - 初创公司：聚焦"为什么做这个"而非"做了多久"
   - 成熟公司：平衡历史沉淀和持续创新

3. **长度控制**：
   - 页面展示：800-1200字
   - 宣传册：500-800字
   - 完整版：可适当扩展"未来"部分

### 写作指南

#### 倒金字塔结构（重要→次要）

1. **开头段**：包含5W1H（Who, What, When, Where, Why, How）
2. **第二段**：关键数据/客户证言
3. **第三段**：技术/产品细节
4. **第四段**：定价/获取方式
5. **结尾**：公司背景

#### 标题公式

```
[公司]发布[产品]，[独特价值]

示例：
✓ Tesla发布Model Y，以$39,000起售价重新定义电动SUV
✓ Slack推出Workflow Builder，无需代码即可自动化工作流
✗ 某公司推出新产品（太泛）
```

#### 数字使用原则

| ✅ 好的用法 | ❌ 避免 |
|-----------|--------|
| 提升35%效率 | 大幅提升效率 |
| 服务10,000+企业 | 服务众多企业 |
| 3分钟完成设置 | 快速完成设置 |

#### 引用规范

- 每篇新闻稿至少2个引用
- 包含：高管1个 + 客户/专家1个
- 引用长度：1-2句话
- 避免空话，要有信息量

### 序列优化建议

1. **打开率优化**：
   - 主题行A/B测试
   - 发送时间优化（通常周二-周四上午10点效果最佳）
   - 个性化主题行（使用收件人姓名）

2. **点击率优化**：
   - 每封邮件只设置1个主要CTA
   - 按钮文案使用动词（如"获取指南"而非"点击这里"）
   - 移动端优化（60%+邮件在手机上打开）

3. **转化优化**：
   - 邮件3后未打开者：尝试不同主题行重发
   - 点击但未转化者：追加1-2封邮件解决异议
   - 转化者：移入"客户培育"序列
