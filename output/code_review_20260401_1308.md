### 2.1 `bad_code_example.py` - 教学用的"反面教材"

**此文件明显是有意编写的问题代码示例，包含多种典型问题：**

| 问题类型 | 具体问题 | 风险等级 |
|----------|----------|----------|
| **安全** | 使用 MD5 哈希密码（可被彩虹表攻击） | 🔴 高 |
| **安全** | `os.system()` 直接执行命令，存在命令注入风险 | 🔴 高 |
| **Bug** | `process_data()` 函数 O(n²) 时间复杂度 | 🟡 中 |
| **Bug** | 邮件分割无异常处理，格式错误会崩溃 | 🟡 中 |
| **代码规范** | 类名使用小驼峰 `userManager` | 🟢 低 |
| **代码规范** | 一行导入多个模块 | 🟢 低 |
| **资源泄漏** | `exportToCSV()` 文件未使用 `with` 语句 | 🟡 中 |

**修复建议：**
```python
# 1. 使用 bcrypt/argon2 替代 MD5
import bcrypt
hashed = bcrypt.hashpw(pwd.encode(), bcrypt.gensalt())

# 2. 使用参数化命令执行
import shutil
shutil.copy('users.json', f'backup_{int(time.time())}.json')

# 3. 修复 O(n²) 算法
def process_data(data_list):
    return list(set([x for x in data_list if data_list.count(x) > 1]))
```

### 2.2 `example_flask_api.py` - 包含多个严重安全漏洞

**⚠️ 此文件包含生产环境不可用的代码！**

| 问题 | 位置 | 风险 |
|------|------|------|
| **SQL注入** | `create_user()`, `get_user()`, `update_user()`, `delete_user()`, `search_users()`, `login()` | 🔴 **严重** |
| **硬编码密钥** | `SECRET_KEY = 'hardcoded_secret_key_12345'` | 🔴 **严重** |
| **密码泄露** | `get_user()` 返回用户密码哈希 | 🔴 **严重** |
| **弱密码哈希** | 使用 MD5 而非 bcrypt/argon2 | 🔴 **高** |
| **缺乏权限验证** | 任何人可修改/删除任意用户 | 🔴 **高** |
| **调试路由暴露** | `/debug/users` 暴露内部数据 | 🔴 **高** |
| **缺乏输入验证** | 直接访问 `data['username']` 无校验 | 🟡 **中** |
| **性能问题** | `get_users()` 无分页，大数据量会崩溃 | 🟡 **中** |

**危险代码示例：**
```python
# SQL注入漏洞！
query = f"INSERT INTO users (username, email, password, age) VALUES ('{username}', '{email}', '{hashed_password}', {age})"

# 修复方案：使用参数化查询
query = "INSERT INTO users (username, email, password, age) VALUES (?, ?, ?, ?)"
g.db.execute(query, (username, email, hashed_password, age))
```

## 5. 修复建议总结

### 立即修复 (P0)
1. **所有 SQL 注入** → 使用参数化查询
2. **移除所有 `eval()`** → 使用 `json.loads()` 或 `ast.literal_eval()`
3. **硬编码密钥** → 使用环境变量或配置文件
4. **密码哈希** → 使用 `bcrypt` 或 `argon2`

### 尽快修复 (P1)
1. **添加输入验证** → 使用 `pydantic` 或 `marshmallow`
2. **添加分页** → 防止大数据量查询
3. **异常处理** → 添加 try-except 块
4. **资源管理** → 使用 `with` 语句

### 建议优化 (P2)
1. **日志配置** → 使用 `logging` 模块替代 `print`
2. **类型注解** → 添加完整类型提示
3. **单元测试** → 为核心功能添加测试
4. **文档** → 添加 docstring
