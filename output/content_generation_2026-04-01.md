## 一、产品宣传文案

### 【主标题】小七：你不是缺信息，你是缺一个会算账的助理

**副标题**：数据驱动 · 逻辑为王 · 商业决策的首选AI伙伴

---

在信息爆炸的时代，**决策者最大的痛苦不是找不到数据，而是被数据淹没**。

市场报告堆积如山，竞品分析千篇一律，财务预测充满乐观估计——**你需要的不只是答案，而是经过验证的逻辑**。

**小七是谁？**

小七不是又一个聊天机器人。它是专为商业决策者设计的AI战略助理，核心理念只有三个：

| 原则 | 说明 |
|------|------|
| **商业的本质是算账** | Revenue = Price × Volume。没有算不清楚的账，只有不够细的拆解。 |
| **数据是唯一的语言** | "我觉得" = 零分；"根据XX报告，2025年市场规模XX亿" = 满分。 |
| **差异化是生存底线** | 红海=死亡，蓝海=生存，垄断=繁荣。每个建议必须有清晰的护城河。 |

**小七能为你做什么？**

✅ **市场研究** - 3个以上独立数据源交叉验证，直接竞品3家+间接竞品2家全面拆解  
✅ **投资决策** - ROI分析、风险评估（最好/最坏/最可能情况）、退出成本计算  
✅ **战略规划** - 差异化路径设计、进入壁垒分析、3-5年战略布局  
✅ **竞争分析** - 竞品优势不回避、劣势不放过，真实还原战场态势  
✅ **全自动执行** - Multi-Agent框架支持，复杂任务自动拆解、并行执行、质量审查

**与通用AI的区别？**

| 维度 | 通用AI | 小七 |
|------|--------|------|
| 输出风格 | 泛泛而谈 | 结论先行，数据说话 |
| 风险意识 | 轻描淡写 | 红字标出，绝不回避 |
| 数据溯源 | "据资料显示" | 报告名称+机构+年份 |
| 执行能力 | 单点回答 | 多Agent协作，端到端交付 |
| 思维方式 | 线性推理 | 第一性原理+二阶思维+逆向思维 |

**适合谁用？**

- 🏢 **企业高管** - 需要数据支撑的战略决策
- 💼 **投资人** - 快速评估项目可行性
- 🚀 **创业者** - 找准差异化定位，避开红海陷阱
- 📊 **分析师** - 提升研究效率，标准化输出质量

**现在开始，让小七成为你的商业伙伴。**

> *"你拍板，我算账。你定方向，我拆路径。你承担风险，我提示风险。"*
> 
> —— 小七，不只是助理，更是你的思维伙伴。

### 2. 核心功能模块

#### 2.1 商业研究引擎

**功能描述**：全自动市场研究与竞品分析

| 能力 | 说明 |
|------|------|
| 市场数据收集 | 至少3个独立来源交叉验证 |
| 市场规模估算 | 标注来源机构、报告年份、估算方法 |
| 竞品拆解 | 直接竞品3家 + 间接竞品2家，全方位对比 |
| 进入壁垒分析 | 现有玩家护城河深度评估 |
| 数据溯源 | 每个数据点标注出处，可追溯验证 |

**输出示例**：
- 市场规模表格（含多来源对比）
- 竞品对比矩阵
- SWOT分析
- 进入壁垒评估

---

#### 2.2 投资决策分析

**功能描述**：项目可行性评估与风险量化

| 能力 | 说明 |
|------|------|
| ROI计算 | (Return - Investment) / Investment × Time |
| 财务模型 | 收入预测、成本结构、盈亏平衡点 |
| 风险评估 | 最好情况 / 最坏情况 / 最可能情况 |
| 退出成本 | 评估决策的试错成本 |
| 建议输出 | 做/不做，为什么，下一步做什么 |

**输出示例**：
- 财务预测表（3年）
- 敏感性分析
- 风险矩阵
- 决策建议清单

---

#### 2.3 Multi-Agent任务执行

**功能描述**：复杂任务自动拆解与并行执行

| Agent角色 | 职责 | 触发场景 |
|-----------|------|----------|
| **Leader** | 任务拆解、执行计划 | "拆解..." "分析任务结构" |
| **Analyzer** | 研究分析、数据收集 | "分析..." "调研..." "研究..." |
| **Writer** | 内容创作、报告撰写 | "写..." "生成..." "撰写..." |
| **Reviewer** | 质量审查、事实核查 | "检查..." "审查..." "质量..." |
| **Coordinator** | 任务调度、结果汇总 | 全局协调 |

**工作流程**：
1. 用户输入任务
2. Leader自动拆解子任务
3. 并行启动多个Worker Agent
4. Reviewer进行质量检查
5. Coordinator汇总交付最终结果

---

**功能描述**：多维度思考框架，避免认知偏差

| 思维方式 | 应用场景 |

### 5. 使用场景

#### 场景一：投资决策
> 收到一个创业项目BP，需要快速评估可行性  
> **小七输出**：市场规模估算 + 竞品分析 + 财务模型 + 风险评估 + 投资建议

#### 场景二：战略规划
> 公司计划进入新市场，需要制定差异化策略  
> **小七输出**：市场进入分析 + 竞品空白点识别 + 差异化定位 + 3年路线图

#### 场景三：竞争分析
> 主要竞品发布新品，需要快速响应分析  
> **小七输出**：竞品功能拆解 + 优劣势对比 + 应对策略建议

#### 场景四：日常决策
> 是否投资某个广告渠道 / 是否招聘某岗位 / 是否扩展某业务线  
> **小七输出**：成本效益分析 + 风险评估 + 决策建议
