### 2. 5-Agent 架构

```
┌─────────────┐
│  用户输入    │
└──────┬──────┘
       ▼
┌─────────────┐
│ Coordinator │ ← 我 (主控)
└──────┬──────┘
       ▼
┌─────────────┐
│    Leader   │ → 拆解任务 → 生成执行计划
└──────┬──────┘
       ▼
┌───────────────────────────────┐
│      并行 Worker Agents        │
│  ┌─────────┐  ┌─────────┐     │
│  │ Analyzer│  │  Writer │     │
│  │ (研究)  │  │ (创作)  │     │
│  └────┬────┘  └────┬────┘     │
│       └─────┬──────┘          │
│             ▼                 │
│       ┌─────────┐             │
│       │ Reviewer│             │
│       │(质检)   │             │
│       └────┬────┘             │
└────────────┼──────────────────┘
             ▼
┌─────────────┐
│ Coordinator │ → 汇总 → 交付用户
└─────────────┘
```

### 3. 自动任务分配

| 用户请求类型 | 自动分配 Agent |
|-------------|---------------|
| "分析...市场" "研究..." | Analyzer |
| "写...报告" "生成...文案" | Writer |
| "检查...代码" "审查..." | Reviewer |
| 复杂多步骤任务 | Leader → 多Agent |

### 4. 质量保障机制

- **自动检查点**: Leader→Coordinator→Reviewer 三级验证
- **质量评分**: Reviewer 1-10分评分，<7分自动重写
- **重试机制**: 失败自动重试 (最多3次)
- **超时处理**: 5分钟默认，复杂任务10分钟

### 5. 输出规范

| Agent | 输出文件 | 格式 |
|-------|---------|------|
| Leader | `leader_plan_*.json` | 执行计划 |
| Analyzer | `analysis_*.md` | 研究报告 |
| Writer | `content_*.md` | 内容文档 |
| Reviewer | `review_*.md` | 质检报告 |
| Final | `final_*.md` | 最终交付 |

## 如何使用

### 方式1: 关键词触发 (自动)
只需在任务描述中包含：
- `"使用 Multi-Agent 模式"`
- `"拆解任务"`
- `"完整分析+报告"`

我会自动调用 Leader 拆解并启动并行 Agents。

### 方式2: 直接请求
```
用户: "帮我分析AI Agent市场"

我自动执行:
1. Leader: 拆解为 市场研究+竞品分析+报告撰写+质量审查
2. Analyzer (x2): 并行研究市场和竞品
3. Writer: 基于研究结果写报告
4. Reviewer: 质量检查
5. Coordinator: 汇总交付
```
