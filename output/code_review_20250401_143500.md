## 🔴 严重问题 (Critical)

### 1. SQL注入漏洞 [CRITICAL-001]

**位置**: `example_flask_api.py:36-38`, `example_flask_api.py:56`, `example_flask_api.py:99`, 多处

**问题代码**:
```python
# example_flask_api.py
query = f"INSERT INTO users (username, email, password, age) VALUES ('{username}', '{email}', '{hashed_password}', {age})"

# database_module.py
query="SELECT * FROM users WHERE name='"+username+"'"
```

**风险**: 攻击者可以通过构造恶意输入执行任意SQL命令，导致数据泄露、篡改或删除。

**修复建议**:
```python
# 使用参数化查询
query = "INSERT INTO users (username, email, password, age) VALUES (?, ?, ?, ?)"
cursor.execute(query, (username, email, hashed_password, age))
```

### 2. 弱密码哈希算法 [CRITICAL-002]

**位置**: `bad_code_example.py:31`, `example_flask_api.py:42`, `database_module.py` (多处)

**问题代码**:
```python
# 使用MD5哈希（已被破解，不安全）
hash=hashlib.md5(pwd.encode()).hexdigest()
hashed_password = hashlib.md5(password.encode()).hexdigest()
```

**风险**: MD5已被证明存在碰撞漏洞，彩虹表攻击可在秒级破解密码。

**修复建议**:
```python
import bcrypt

# 使用bcrypt进行密码哈希（自动加盐）
hashed = bcrypt.hashpw(password.encode('utf-8'), bcrypt.gensalt())
# 验证
if bcrypt.checkpw(password.encode('utf-8'), hashed):
    # 密码正确
```

### W-001: O(n²) 性能问题

**位置**: `bad_code_example.py:107-112`

```python
# 问题代码
result=[]
for i in range(len(data_list)):
    for j in range(len(data_list)):
        if i!=j:
            if data_list[i]==data_list[j]:
                result.append(data_list[i])
```

**优化后**:
```python
from collections import Counter

def process_data(data_list):
    """O(n) 复杂度"""
    counts = Counter(data_list)
    return [item for item, count in counts.items() if count > 1]
```

def get_users():

    per_page = request.args.get('per_page', 20, type=int)
    per_page = min(per_page, 100)  # 限制最大页大小
