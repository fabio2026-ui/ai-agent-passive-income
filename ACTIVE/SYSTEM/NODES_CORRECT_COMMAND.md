✅ 正确命令结构
========================================

## 1. 查看帮助

```bash
openclaw nodes --help
```

## 2. 可用命令

| 命令 | 功能 |
|------|------|
| `openclaw nodes list` | 列出已配对的节点 |
| `openclaw nodes pending` | 列出待批准的配对请求 |
| `openclaw nodes approve <id>` | 批准配对请求 |
| `openclaw nodes run <node> <cmd>` | 在节点上执行命令 |
| `openclaw nodes status` | 查看节点连接状态 |

========================================
🤔 如何发起配对？

========================================

从截图看，`openclaw nodes` 命令是**管理**配对，不是**发起**配对。

发起配对可能的方式：

### 方式1: 通过Gateway Web界面
```bash
openclaw dashboard
```
这会打开控制面板，可能有配对功能。

### 方式2: 检查pending请求
```bash
openclaw nodes pending
```
如果有待处理的请求，可以用approve批准。

### 方式3: 查看OpenClaw文档
节点配对可能需要：
1. 在Mac上启动某种发现服务
2. 在服务器上检测到并发起配对
3. Mac端确认
