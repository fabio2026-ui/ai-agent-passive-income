# 小七产品质量评估与筛选标准
# Product Quality Assessment & Screening Criteria
# 原则: 宁可少发布，也要保证每个产品都有竞争力

## 🎯 核心原则

### 1. 诚实原则
- ❌ **绝不**: 夸大产品能力，忽悠用户
- ❌ **绝不**: 为了数量牺牲质量
- ❌ **绝不**: 发布"半成品"或"概念产品"
- ✅ **必须**: 产品确实能解决真实问题
- ✅ **必须**: 数据支撑竞争力 claims
- ✅ **必须**: 差异化逻辑清晰

### 2. 竞争力标准
一个产品必须有以下至少2项才能发布：

| 维度 | 标准 | 权重 |
|------|------|------|
| **差异化** | 相比竞品有独特价值 | 30% |
| **质量** | 无明显bug，体验流畅 | 25% |
| **需求** | 解决真实痛点，不是伪需求 | 25% |
| **变现** | 有清晰的商业模式 | 10% |
| **可维护** | 长期维护成本可控 | 10% |

### 3. 发布门槛
**必须全部满足才发布**：
- [ ] 核心功能完整可用
- [ ] UI/UX达到行业平均水平
- [ ] 有至少1个竞品对比优势
- [ ] 用户引导/帮助文档完善
- [ ] 错误处理完善
- [ ] 性能评分 >= 80/100

## 📝 发布检查清单

### 每个产品发布前必须检查：

**功能性** (必须100%)
- [ ] 核心功能可用
- [ ] 无严重bug
- [ ] 错误处理完善
- [ ] 性能达标

**用户体验** (必须80%+)
- [ ] UI整洁
- [ ] 操作流畅
- [ ] 有引导/教程
- [ ] 移动端适配

**竞争力** (必须明确)
- [ ] 相比竞品有优势
- [ ] 差异化逻辑清晰
- [ ] 能解决真实问题
- [ ] 不是"me too"产品

**商业准备** (必须完整)
- [ ] 定价清晰
- [ ] 支付流程测试
- [ ] 有营销素材
- [ ] 支持渠道准备

**诚实声明** (必须包含)
- [ ] 产品限制说明
- [ ] 非医疗/法律免责声明 (如适用)
- [ ] 数据隐私说明

---

## 🚫 禁止发布的情况

1. **功能不完整**: 核心功能有bug或缺失
2. **体验糟糕**: 加载慢、卡顿、UI混乱
3. **无差异化**: 和竞品一模一样
4. **伪需求**: 解决不存在的问题
5. **维护困难**: 长期维护成本过高
6. **法律风险**: 无免责声明的医疗/法律建议
