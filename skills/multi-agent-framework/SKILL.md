# Multi-Agent Framework 2.0

## 架构设计

一个主控 Agent（我）作为 **Coordinator**，负责解析任务并分发子任务给 4 个 Worker Agent。

## Agent 职责定义

### 1. Leader Agent (任务理解)
- **输入**: 用户原始需求
- **职责**: 
  - 拆解任务为可执行的子任务
  - 确定子任务优先级和依赖关系
  - 生成任务执行计划
- **输出**: 结构化任务清单 (JSON格式)

### 2. Analyzer Agent (研究分析)
- **输入**: Leader 分配的研究型子任务
- **职责**:
  - 数据收集和验证
  - 竞品分析
  - 市场研究
  - 技术调研
- **输出**: 结构化分析报告 (含数据来源)

### 3. Writer Agent (内容创作)
- **输入**: Leader 分配的创作型子任务 + Analyzer 的研究结果
- **职责**:
  - 撰写报告、文案、文档
  - 数据可视化建议
  - 内容格式化
- **输出**: 符合要求的完整内容

### 4. Reviewer Agent (质量审查)
- **输入**: Writer 产出的内容
- **职责**:
  - 事实准确性核查
  - 逻辑一致性检查
  - 格式规范性审查
  - 质量评分 (1-10)
- **输出**: 审查报告 + 修改建议

### 5. Coordinator (主控)
- 就是我，当前对话中的 Agent
- **职责**:
  - 接收用户任务
  - 调用 Leader Agent 拆解任务
  - 并行调度 Worker Agents
  - 汇总结果并交付

## 任务分配策略

### 任务类型识别
| 任务类型 | 分配 Agent | 示例 |
|---------|-----------|------|
| 研究分析 | Analyzer | 竞品分析、市场调研 |
| 内容创作 | Writer | 撰写报告、营销文案 |
| 代码审查 | Reviewer | 代码质量检查 |
| 质量验证 | Reviewer | 事实核查、逻辑检查 |
| 复杂拆解 | Leader | 大型项目拆解 |

### 依赖关系处理
```json
{
  "tasks": [
    {"id": "T1", "type": "analysis", "agent": "Analyzer", "depends_on": []},
    {"id": "T2", "type": "writing", "agent": "Writer", "depends_on": ["T1"]},
    {"id": "T3", "type": "review", "agent": "Reviewer", "depends_on": ["T2"]}
  ]
}
```

## 通信协议

### 子任务消息格式
```json

  "type": "analysis|writing|review",
  "input": "任务描述",
  "context": "相关上下文",
  "output_format": "要求的输出格式",
  "deadline": "预期完成时间"
}
```

### 结果消息格式
```json
{
  "task_id": "uuid",
  "status": "completed|failed",
  "result": "结果内容",
  "quality_score": 8.5,
  "issues": ["发现的问题"],
  "suggestions": ["改进建议"]
}
```

## 错误处理机制

### 重试策略
- **Analyzer**: 数据源失效时，尝试备选数据源 (最多3次)
- **Writer**: 内容不符合要求时，根据 Reviewer 反馈重写 (最多2次)
- **Reviewer**: 质量评分 < 7 时，退回 Writer 修改

### 超时处理
- 默认超时: 5分钟
- 复杂任务: 10分钟
- 超时后标记为失败，由 Coordinator 决定是否重试或降级

## 质量保障

### 自动检查点
1. Leader 拆解后 → Coordinator 验证合理性
2. Analyzer 完成后 → 数据源可追溯性检查
3. Writer 完成后 → Reviewer 质量评分
4. Reviewer 评分 < 7 → 自动退回重写

### 人工介入触发条件
- 连续2次重试失败
- Reviewer 发现事实性错误
- 用户明确要求人工确认

## 性能优化

### 并行化
- 无依赖的任务并行执行
- 最大并行数: 3 (避免资源竞争)

### 缓存机制
- 研究结果缓存 24h
- 常用数据缓存 (市场规模、竞品信息等)

### 增量更新
- 已有报告更新时，只修改变更部分
- 保留历史版本

## 使用示例

### 示例: 生成行业报告
```
用户: "帮我写一份2026年AI Agent行业趋势报告"

Coordinator → Leader
"拆解为: 1)市场研究 2)竞品分析 3)趋势预测 4)报告撰写"

并行执行:
- Analyzer: 收集市场数据、竞品信息
- Analyzer: 研究技术趋势

依赖执行:
- Writer: 基于分析结果撰写报告
- Reviewer: 质量检查

Coordinator: 汇总 → 交付完整报告
```
