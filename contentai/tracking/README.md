# ContentAI 用户获取追踪系统

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    数据采集层                                 │
│  UTM参数 → 用户行为 → 转化事件 → 收入数据                      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    数据存储层                                 │
│  用户表 → 渠道表 → 转化表 → 收入表                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    分析看板层                                 │
│  渠道效果 | 转化率 | ROI | 趋势分析                            │
└─────────────────────────────────────────────────────────────┘
```

## 核心指标

| 指标 | 说明 | 计算公式 |
|------|------|----------|
| UV | 独立访客数 | 去重用户ID计数 |
| PV | 页面浏览量 | 页面访问次数 |
| CTR | 点击率 | 点击数 / 曝光数 × 100% |
| CR | 转化率 | 转化数 / 访问数 × 100% |
| CAC | 获客成本 | 渠道花费 / 新用户数 |
| LTV | 用户生命周期价值 | 总收入 / 总用户数 |
| ROI | 投资回报率 | (收入 - 成本) / 成本 × 100% |
| ARPU | 每用户平均收入 | 总收入 / 活跃用户数 |

## 数据表关系

```
users (1) ──────< (N) conversions (N) ──────> (1) revenue
   │
   └────────────< (N) channel_stats
```

## 使用流程

1. **生成追踪链接**：使用 `utm-generator.html`
2. **分发内容**：在各渠道发布带UTM参数的内容
3. **数据采集**：自动记录用户行为和转化
4. **查看报表**：打开 `dashboard.html` 查看实时数据
5. **优化决策**：根据数据分析调整渠道策略
