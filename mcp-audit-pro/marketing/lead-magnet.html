Top 10 MCP Security Mistakes

Based on auditing 30+ open-source MCP servers. These are the most common vulnerabilities we found — and how to fix them.

1

Command Injection via exec()

Passing user input directly to shell commands without sanitization.

2

Path Traversal in File Operations

Allowing ../ sequences to escape the intended directory.

3

Hardcoded API Keys

Credentials committed to source code repositories.

4

Running Containers as Root

Dockerfiles using USER root or chmod 777 everywhere.

5

SSRF via Unrestricted Fetch

Allowing the MCP server to request any URL, including internal services.

6

Missing Input Validation

Trusting AI-generated tool arguments without validation.

7

Insecure Deserialization

Passing untrusted data to eval(), JSON.parse(), or pickle.loads().

8

Overly Broad Permissions

Granting filesystem or network access without scope restrictions.

9

No Rate Limiting

Allowing infinite tool calls that can exhaust resources or budgets.

10

No Security Headers

Missing CORS, CSP, and other HTTP security controls on MCP transports.
