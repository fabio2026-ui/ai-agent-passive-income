Frequently Asked Questions

Everything you need to know about MCP security audits and our services.

What is MCP? +
MCP stands for Model Context Protocol. It's an open standard that lets AI agents (like Claude, Cursor, and Windsurf) connect to external tools, databases, and APIs. Think of it as "plugins for AI" that allow these agents to interact with your systems.
Why do MCP servers need security audits? +
MCP servers act as a bridge between AI agents and your systems. A vulnerable MCP server can let an AI agent:
  • Execute arbitrary commands on your machine
  • Read files it shouldn't have access to
  • Access internal APIs and cloud metadata
  • Leak hardcoded credentials
When an AI agent has direct system access, every security bug becomes critical.
What vulnerabilities do you check for? +
We analyze MCP servers for:
  • Command Injection — Unsanitized user input passed to shell commands
  • Path Traversal — File operations without path validation
  • Privilege Escalation — Running with unnecessary root/sudo access
  • Credential Exposure — Hardcoded API keys, tokens, and passwords
  • Unrestricted Network Access — SSRF via arbitrary URL fetching

What do I get in a professional audit report? +
Our professional audit reports include:

  • Code examples showing vulnerable vs. secure patterns
  • PDF format suitable for sharing with stakeholders

Yes, for professional audits we can analyze private repositories. We sign NDAs upon request and delete all cloned code after the audit is complete. Your intellectual property is fully protected.
