## 一、顶级Pitch Deck案例分析

### 1. Airbnb (2009) - 传奇Seed Deck
**核心特点：**
- **配色**: 深红(#FF5A5F) + 白 + 深灰
- **字体**: 简洁无衬线，大标题
- **结构**: Problem → Solution → Market → Product → Business Model → Traction
- **亮点**: 用"酒店体验差"三图直击痛点

**可复用模式：**
- 痛点页用对比图（Before/After）
- 市场页用TAM/SAM/SOM三层漏斗
- 用真实用户评价截图代替文字

### 2. Dropbox (2007) - 简洁典范
**核心特点：**
- **配色**: 深蓝(#0061FF) + 白 + 少量荧光绿
- **字体**: 超大标题(60pt+)，正文极少
- **结构**: Problem → Solution → Demo → Business Model
- **亮点**: 3分钟视频代替文字演示

**可复用模式：**
- 一句话描述产品（3秒内理解）
- 技术细节隐藏，展示价值
- 创始团队用头像+一句话背景

### 3. Canva (2013) - 视觉冲击力
**核心特点：**
- **配色**: 紫(#7B2CBF) + 青(#00C4CC) + 粉(#FF6B9D)
- **字体**: 混排（标题衬线+正文无衬线）
- **结构**: Story → Problem → Solution → Market → Traction
- **亮点**: 用真实设计作品做背景

**可复用模式：**
- 封面用全屏产品截图
- 数据用大号数字+渐变背景
- 时间线用视觉化路径

### 4. Stripe (2010) - 开发者美学
**核心特点：**
- **配色**: 紫(#635BFF) + 深蓝(#0A2540) + 青绿
- **字体**: 等宽代码感 + 现代无衬线
- **结构**: Problem → Solution → Demo → API → Customers
- **亮点**: 代码片段作为设计元素

**可复用模式：**
- 技术产品用代码片段展示
- 深色背景+荧光强调色
- 客户logo墙展示牵引力

### 5. Figma (2015) - 协作感
**核心特点：**
- **配色**: 紫红(#F24E1E) + 黑 + 白
- **字体**: 极粗标题 + 细正文对比
- **结构**: Problem → Solution → Product → Community → Future
- **亮点**: 用协作场景图（多人光标）

**可复用模式：**
- 展示"多人协作"的社交证明
- 社区数据（用户数、文件数）前置
- 未来愿景用概念图

## 二、色彩理论速查表

### 主色选择矩阵

| 行业/类型 | 推荐主色 | 心理暗示 | 代表品牌 |
|-----------|----------|----------|----------|
| 金融科技 | 深蓝 #0066FF | 信任、专业 | Stripe, PayPal |
| 医疗健康 | 青绿 #00C4B4 | 安全、生命 | Teladoc, Ro |
| 消费品牌 | 暖红 #FF5A5F | 热情、冲动 | Airbnb, Coca-Cola |
| 企业服务 | 紫 #635BFF | 创新、智慧 | Slack, Notion |
| 社交/内容 | 粉/橙 #FF6B6B | 活力、年轻 | Instagram, TikTok |
| 环保/可持续 | 绿 #00D26A | 自然、成长 | Allbirds, Beyond Meat |

### 对比度规则（WCAG 2.1）
- **AA级**: 4.5:1（正文），3:1（大标题）
- **AAA级**: 7:1（正文），4.5:1（大标题）
- **工具**: WebAIM Contrast Checker, Stark

### 配色方案类型

**单色方案 (Monochromatic)**
- 同色相，不同明度/饱和度
- 最安全，适合严肃场合
- 例：蓝 #0066FF → #3385FF → #66A3FF

**类比方案 (Analogous)**
- 色轮相邻色（如蓝-青-绿）
- 和谐统一，适合自然主题
- 例：Stripe的蓝-青-绿

**互补方案 (Complementary)**
- 色轮对面色（如红-绿，蓝-橙）
- 高对比，适合行动号召
- 例：Airbnb红 + 青绿按钮

**分裂互补 (Split Complementary)**
- 主色 + 对面两侧色
- 对比强但不刺眼
- 例：蓝主色 + 黄橙/红橙强调

## 三、字体搭配推荐

### 经典组合

| 标题 | 正文 | 风格 | 适用 |
|------|------|------|------|
| Inter | Inter | 极简现代 | 科技、SaaS |
| Playfair Display | Inter | 优雅专业 | 金融、咨询 |
| Archivo Black | Source Sans Pro |  bold冲击 | DTC品牌 |
| Space Grotesk | Inter | 几何未来 | AI、Web3 |
| Crimson Pro | Lato | 人文温暖 | 教育、健康 |
| JetBrains Mono | Inter | 开发者 | 技术产品 |

### 字号系统（Pitch Deck）

```
Cover标题:     72-96px  (font-bold)
章节标题:      48-64px  (font-bold)  
副标题:        24-32px  (font-normal)
正文:          18-20px  (font-normal)
标注/来源:     14-16px  (opacity-60)
```

### 排版原则
1. **大>多**: 宁可少字，也要字号大
2. **留白是金**: 四周至少留15%空白
3. **对齐一致**: 左对齐最保险
4. **层次清晰**: 标题→副标题→正文，大小对比明显

## 四、数据可视化最佳实践

### 1. 增长曲线
- 用面积图比折线图更有冲击力
- 起点归零，避免截断Y轴
- 关键节点标注（融资、产品发布）

### 2. 市场份额
- 饼图最多4-5个分类
- 用横向条形图替代（更易读）
- 你的份额用强调色，其他用灰色

### 3. 财务表格
- 关键数字用强调色+加粗
- 对齐：文字左对齐，数字右对齐
- 最多4列，避免拥挤

### 4. 时间线
- 用图标+短句，不用长描述
- 垂直或水平都可以，保持一致
- 当前节点高亮，过去节点灰色

## 五、一页设计检查清单

### 封面页 ✅
- [ ] 项目名一眼可见（72px+）
- [ ] 一句话定位清晰
- [ ] 3个核心数字
- [ ] 融资阶段标注

### Problem页 ✅
- [ ] 痛点用大图或数字可视化
- [ ] 不超过2个痛点
- [ ] 每个痛点一句话解释

### Solution页 ✅
- [ ] 产品截图或示意图
- [ ] 核心价值主张一句话
- [ ] 3个关键优势（图标+短句）

### Market页 ✅
- [ ] TAM数字超大展示
- [ ] 数据来源标注
- [ ] 细分市场规模（可选）

### Business Model页 ✅
- [ ] 收入来源占比可视化
- [ ] 单位经济（CAC/LTV/_margin）
- [ ] 与行业标准对比（可选）

### Traction页 ✅
- [ ] 增长曲线或里程碑
- [ ] 客户引用（带照片/公司名）
- [ ] 关键指标（MRR/用户/增长率）

### Team页 ✅
- [ ] 创始人照片+一句话背景
- [ ] 之前成就/退出经历
- [ ] 团队规模/结构

### Financials页 ✅
- [ ] 3-5年预测表格
- [ ] 关键假设脚注
- [ ] 达到盈亏平衡时间点

### Closing页 ✅
- [ ] 融资金额醒目
- [ ] 资金用途简要说明
- [ ] 联系方式
- [ ] 强行动号召（不是"谢谢"）
